APIs for Quiz Game
format_list_bulletedDescription
edit Submission
descriptionBackground

Design and Develop APIs for a Quiz. You will have to perform micro-tasks like creating quizzes, questions, listing them, etc as mentioned below.

assignmentExercises

  1. Get a quiz

    GET /api/quiz/:quiz_id 

    Response Code 200 : Expected response should be of the form: 

    {
    "id": ,                     ?-------------- number
    "name": "",
    "description": ""
} 

    Response Code 404 (Not Found) : Expected response should be of the form: 

    {}                              ?--------- empty response 

    Response Code 400 (failure) : Expected response should be of the form: 

    {
    "status": "failure",
    "reason": "explanation"     ?--------- explanation can be any message
}

  2. Create a quiz

    POST /api/quiz/ 

    Expected request should be of the form 

    {
    "name": "",                 ?------------- string (required)
    "description": ""           ?------------- string (required)
} 

    Response Code 201 : Expected response should be of the form: 

    {
    "id": ,                     ?-------------- number
    "name": "",
    "description": ""
} 

    Response Code 400 (failure) : Expected response should be of the form: 

    {
    "status": "failure",
    "reason": "explanation"     ?--------- explanation can be any message
}

  3. Get a Question

    Fetch a question by id 

    GET /api/questions/:question_id

    Response Code 200 : Expected response should be of the form: 

    {
    "id": ,                     ?--------- number
    "name": "",
    "options": "",
    "correct_option": ,
    "quiz": ,
    "points": ,
} 

    Response Code 404 (Not found) : Expected response should be of the form: 

    {}

    Response Code 400 (failure) : Expected response should be of the form: 

     {
    "status": "failure",
    "reason": "explanation"     ?--------- explanation can be any message
}

  4. Create a Question

    POST /api/questions/ 

    Expected request should be of the form: 

    {
    "name": "",                 ?--------- string (required)
    "options": "",              ?--------- string (comma separated values, required)
    "correct_option": ,         ?--------- integer (required)
    "quiz": ,                   ?--------- foreign key for quiz (required)
    "points":                   ?--------- integer (required)
} 

    Response Code 201 : Expected response should be of the form: 

    {
    "id": ,                     ?-------- Should be a number 
    "name": "",
    "options": "",
    "correct_option": ,
    "quiz": ,
    "points":
} 

    Response Code 400 (failure) : Expected response should be of the form: 

    {
    "status": "failure",
    "reason": "explanation"     ?--------- explanation can be any message
}

  5. Get all questions for a quiz

    GET /api/quiz-questions/:quiz_id 

    Response Code 200 : Expected response should be of the form: 

    {
    "name": "sample_quiz",
    "description": "sample text",
    "questions" :
        [{
            "id": ,             ?------- should be a number 
            "name": "",
            "options": "",
            "correct_option": ,
            "quiz": ,
            "points": ,
        }]
}

    Response Code 404 (Not found) : Expected response should be of the form: 

    {}

  • In case you need to store data in a database, then please use an in-memory relational/non-relational database like HSQLDB, SQLite, in-memory MongoDB, etc for performing operations on the data.

  • Please clean your local database before running any test case (in Step 2. Execute Test Cases).

  • Please note that every test case has a few sub-test cases inside. These sub-test cases are run in sequential order. So, when the "Run" button (in Step 2.) is clicked for executing a test case, then "if one of the sub-test cases fail, then the subsequent sub-test cases might also fail", as a result of which templatised fields like "$id", "$id2" can be seen when debugging the code.

  • Please note that you can submit the solution multiple times.
